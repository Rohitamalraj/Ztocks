## ❌ Contracts to REMOVE (4 deprecated)

### 1. SynthToken.sol ❌
**Reason**: Replaced by ConfidentialSynthToken.sol (ERC7984)

**Old approach**:
```solidity
// SynthToken.sol - Plaintext ERC20
contract SynthToken is ERC20, Ownable {
    function mint(address to, uint256 amount) external onlyVault {
        _mint(to, amount); // ❌ Plaintext balance
    }
}
```

**New approach**:
```solidity
// ConfidentialSynthToken.sol - Encrypted ERC7984
contract ConfidentialSynthToken is ERC7984, Ownable2Step {
    function mint(address to, euint64 encryptedAmount) external onlyVault {
        _mint(to, encryptedAmount); // ✅ Encrypted balance
    }
}
```

**Impact**: None (not used in current flow)

---

### 2. ConfidentialSynthVault.sol ❌
**Reason**: Replaced by ConfidentialSynthVaultFHE.sol (full FHE implementation)

**Old approach**:
```solidity
// ConfidentialSynthVault.sol - Partial FHE
contract ConfidentialSynthVault {
    struct Position {
        uint256 collateral;  // ❌ Plaintext
        uint8 leverage;      // ❌ Plaintext
        bool isLong;         // ❌ Plaintext
    }
}
```

**New approach**:
```solidity
// ConfidentialSynthVaultFHE.sol - Full FHE
contract ConfidentialSynthVaultFHE {
    struct EncryptedPosition {
        euint64 collateral;  // ✅ Encrypted
        euint8 leverage;     // ✅ Encrypted
        ebool isLong;        // ✅ Encrypted
    }
}
```

**Impact**: None (not used in current flow)

---

### 3. TierRegistry.sol ❌
**Reason**: Replaced by ConfidentialTierManager.sol (encrypted tiers)

**Old approach**:
```solidity
// TierRegistry.sol - Plaintext tiers
contract TierRegistry {
    mapping(address => uint8) public userTier; // ❌ Plaintext
}
```

**New approach**:
```solidity
// ConfidentialTierManager.sol - Encrypted tiers
contract ConfidentialTierManager {
    mapping(address => euint8) private encryptedTier; // ✅ Encrypted
}
```

**Impact**: None (not used in current flow)

---

### 4. SynthOracle.sol ❌
**Reason**: Not used (prices fetched from backend API)

**Current approach**:
- Backend fetches prices from Finnhub + Bybit
- Frontend passes price as parameter to `openPosition()`
- No on-chain oracle needed

**Why not used**:
- Real-time stock prices require off-chain data
- Chainlink doesn't support all stocks
- Backend API is more flexible and cost-effective

**Impact**: None (never integrated)

## 📋 Cleanup Checklist

### Step 1: Backup (Safety First)
```bash
# Create backup branch
git checkout -b backup-before-cleanup
git push origin backup-before-cleanup
```

### Step 2: Remove Deprecated Contracts
```bash
# Remove old contract files
rm contracts/contracts/SynthToken.sol
rm contracts/contracts/ConfidentialSynthVault.sol
rm contracts/contracts/TierRegistry.sol
rm contracts/contracts/SynthOracle.sol
```

### Step 3: Remove Unused Mocks
```bash
# Remove unused mock files
rm contracts/contracts/mocks/MockHSP.sol
rm contracts/contracts/mocks/MockKycSBT.sol
rm contracts/contracts/mocks/MockSupraOracle.sol
```

### Step 4: Remove Unused Interfaces
```bash
# Remove unused interface files
rm contracts/contracts/interfaces/IKycSBT.sol
rm contracts/contracts/interfaces/ISupraPullOracle.sol
```

### Step 5: Update Imports
Check if any remaining contracts import the removed files:
```bash
# Search for imports of removed contracts
grep -r "SynthToken.sol" contracts/contracts/
grep -r "ConfidentialSynthVault.sol" contracts/contracts/
grep -r "TierRegistry.sol" contracts/contracts/
grep -r "SynthOracle.sol" contracts/contracts/
```

### Step 6: Recompile
```bash
cd contracts
npm run compile
```

### Step 7: Run Tests
```bash
npm test
```

### Step 8: Update Documentation
- Update README.md to remove references to old contracts
- Update CONTRACT_ARCHITECTURE.md
- Update deployment scripts if needed

## ✅ Benefits of Cleanup

1. **Simpler Codebase**
   - Fewer files to maintain
   - Less confusion about which contracts to use
   - Easier onboarding for new developers

2. **Faster Compilation**
   - Fewer contracts to compile
   - Reduced compilation time

3. **Clearer Architecture**
   - Only production-ready contracts remain
   - Clear separation: core contracts vs mocks

4. **Reduced Gas Costs**
   - No accidental deployment of deprecated contracts
   - Smaller deployment scripts

5. **Better Documentation**
   - Documentation matches actual codebase
   - No references to unused contracts
